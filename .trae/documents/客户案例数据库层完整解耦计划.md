# 客户案例模块数据库层完整解耦计划

## 目标

将案例模块从 `eb_article` + `eb_article_case_extend` 双表 JOIN 架构，改为独立的单表 `eb_case` 架构，实现数据库层、Mapper层、Service层的完整解耦。同时更新前后端 Controller 使其调用新的 CaseService。

---

## 一、当前架构 vs 目标架构

### 当前（有问题）
```
eb_article (主表)  ← 与新闻模块共用
    ↓ article_id
eb_article_case_extend (扩展表)
    ↓
ArticleService + ArticleCaseExtendService
    ↓
CaseController (但注入的是 ArticleService)
```

### 目标（解耦后）
```
eb_case (独立单表，合并主表+扩展表所有字段)
    ↓
CaseDao → CaseMapper.xml
    ↓
CaseService + CaseServiceImpl
    ↓
CaseController (注入 CaseService)
```

## 四、详细实现步骤

### 步骤 1：创建 `crmeb/docs/sql/case.sql`

新建 `eb_case` 单表，合并以下字段：
- 从 `eb_article` 继承：`id`, `title`, `author`, `image_input`, `synopsis`, `share_title`, `share_synopsis`, `content`, `visit`, `sort`, `status`, `hide`, `is_hot`, `is_banner`, `create_time`, `update_time`
- 从 `eb_article_case_extend` 继承：`project_name`, `industry_category`, `region_category`, `project_type`, `project_address`, `project_period`, `project_background`, `supply_products`, `implementation_result`, `product_ids`, `cover_images`
- 新增索引：`industry_category`, `region_category`, `project_type`

同时包含数据迁移 SQL：
```sql
INSERT INTO eb_case (...) SELECT ... FROM eb_article a INNER JOIN eb_article_case_extend e ON a.id = e.article_id;
```

### 步骤 2：创建 `Case.java` 实体类

**路径**：`crmeb-common/src/main/java/com/zbkj/common/model/case/Case.java`

合并 `Article` 和 `ArticleCaseExtend` 所有字段到单表实体，使用 `@TableName("eb_case")` 标注。

注意：字段名遵循 Java 驼峰命名，MyBatis-Plus 自动映射到下划线数据库字段。

### 步骤 3：创建 `CaseRequest.java`

**路径**：`crmeb-common/src/main/java/com/zbkj/common/request/CaseRequest.java`

合并原 `ArticleRequest`（文章基础字段）+ `ArticleRequestCaseExtend`（案例扩展字段）的所有字段，用于新增/更新案例。

### 步骤 4：创建 `CaseSearchRequest.java`

**路径**：`crmeb-common/src/main/java/com/zbkj/common/request/CaseSearchRequest.java`

后台案例搜索参数：`keywords`(标题关键词)、`industryCategory`、`regionCategory`、`projectType`。

### 步骤 5：创建 `CaseListResponse.java`

**路径**：`crmeb-common/src/main/java/com/zbkj/common/response/CaseListResponse.java`

案例列表响应，合并 `ArticleVo`（文章列表VO）的关键字段 + 案例扩展字段（`projectName`, `industryCategory`, `regionCategory`, `projectType`）。

### 步骤 6：创建 `CaseDetailResponse.java`

**路径**：`crmeb-common/src/main/java/com/zbkj/common/response/CaseDetailResponse.java`

案例详情响应，包含：案例所有基础字段 + 扩展字段 + 解析后的关联商品列表 `List<Product>`。

### 步骤 7：创建 `CaseDao.java`

**路径**：`crmeb-service/src/main/java/com/zbkj/service/dao/CaseDao.java`

```java
public interface CaseDao extends BaseMapper<Case> {
}
```
继承 MyBatis-Plus BaseMapper，获得内置 CRUD 方法。如有自定义查询（如去重行业分类），通过 Mapper XML 实现。

### 步骤 8：创建 `CaseMapper.xml`

**路径**：`crmeb-service/src/main/resources/mapper/case/CaseMapper.xml`

包含：
- `BaseResultMap`：字段映射
- 自定义查询：`getDistinctIndustryCategories`（原在 ArticleCaseExtendService 中，现在直接在 case 表上查询）

### 步骤 9：创建 `CaseService.java`

**路径**：`crmeb-service/src/main/java/com/zbkj/service/service/CaseService.java`

接口方法：
- `PageInfo<CaseListResponse> getAdminList(CaseSearchRequest request, PageParamRequest pageParamRequest)` — 后台分页列表
- `Case getDetail(Integer id)` — 查询详情（基础信息）
- `CaseDetailResponse getCaseDetail(Integer id)` — 前台案例详情（含关联商品）
- `PageInfo<CaseListResponse> getCaseFrontList(String cid, CaseFrontSearchRequest request, PageParamRequest pageParamRequest)` — 前台案例列表
- `Boolean create(CaseRequest request)` — 新增
- `Boolean update(Integer id, CaseRequest request)` — 更新
- `Boolean delete(Integer id)` — 删除
- `Boolean bindProduct(Integer id, Integer productId)` — 绑定商品
- `List<String> getDistinctIndustryCategories()` — 去重行业分类

### 步骤 10：创建 `CaseServiceImpl.java`

**路径**：`crmeb-service/src/main/java/com/zbkj/service/service/impl/CaseServiceImpl.java`

实现 `CaseService` 所有方法，核心逻辑参考原 `ArticleServiceImpl` 中 case 相关方法：
- `getAdminList`：使用 MyBatis-Plus LambdaQueryWrapper 构建关键词+行业+区域筛选，分页查询 `eb_case`
- `getCaseDetail`：查询 case + 解析 `product_ids` JSON 关联商品
- `getCaseFrontList`：筛选 + 分页
- `create`：直接插入 `eb_case`（单表，无需事务处理扩展表）
- `update`：直接更新 `eb_case`
- `delete`：直接删除
- `bindProduct`：查询 case 的 `product_ids` JSON，追加去重后更新
- `getDistinctIndustryCategories`：`SELECT DISTINCT industry_category FROM eb_case WHERE industry_category != ''`

### 步骤 11：修改 Admin 端 `CaseController.java`

**改动要点**：
- 注入 `CaseService` 替代 `ArticleService` + `ArticleCaseExtendService`
- `getList` 方法：参数类型 `CaseSearchRequest`，响应类型 `CaseListResponse`
- `save` 方法：参数类型 `CaseRequest`（单表请求，无需 caseInfo 嵌套）
- `delete` 方法：调用 `caseService.delete(id)`
- `update` 方法：参数类型 `CaseRequest`
- `info` 方法：返回 `Case` 实体
- `infoWithCase` 方法：返回 `CaseDetailResponse`（无需 Map 包装，单表查询）
- `bindProduct` 方法：调用 `caseService.bindProduct(id, productId)`
- `getDistinctIndustryCategories`：新增接口返回去重行业分类

### 步骤 12：修改 Front 端 `CaseController.java`

**改动要点**：
- 注入 `CaseService` 替代 `ArticleService` + `ArticleCaseExtendService`
- `getCaseList`：参数类型 `CaseSearchRequest`（去掉 `cid` 路径变量，改为可选查询参数，或保留作为前端过滤参数）
- `caseInfo`：返回 `CaseDetailResponse`
- `getCaseIndustryCategories`：调用 `caseService.getDistinctIndustryCategories()`

### 步骤 13：修改 `admin/src/api/case.js`

**改动要点**：
- `getCaseList`：请求参数字段对齐 `CaseSearchRequest`（`industryCategory`, `regionCategory`, `projectType`, `keywords`）
- `saveCase` / `updateCase`：请求 body 平铺所有字段（不再嵌套 `caseInfo`），字段名对齐 `CaseRequest`（驼峰转下划线由框架自动处理）
- `getCaseDetail`：改为 `/admin/case/info`，返回平铺的 `Case` 对象

### 步骤 14：修改 `admin/src/views/content/case/edit.vue`

**改动要点**：
- 将 `pram.caseInfo.*` 子对象展开为 `pram.*` 平铺字段
- 删除"文章分类"选择器（case 不再依赖文章分类体系）
- 调整 API 调用，适配新的 `saveCase` / `updateCase` 请求体格式
- `getInfo()` 返回的是平铺的 Case 对象，不再需要从 `data.article` + `data.caseInfo` 分拆

### 步骤 15：修改 `admin/src/views/content/case/list.vue`

**改动要点**：
- 移除"文章分类"筛选下拉框（case 不使用文章分类体系）
- 移除分类树数据加载 (`handlerGetTreeList`)
- 调整搜索参数为 `CaseSearchRequest` 字段
- 列表列：保留 ID、图片、标题、行业分类、区域分类、项目类型、项目名称、浏览量、更新时间、操作

### 步骤 16：修改 `app/api/api.js`

**改动要点**：
- `getCaseList`：`cid` 改为可选参数（或改为 `categoryId`），参数对齐 `CaseFrontSearchRequest`

## 七、注意事项

- 原 `eb_article`、`eb_article_case_extend` 表**不删除**
- 原 `ArticleService` 中 case 相关方法**暂不移除**，避免影响其他未知引用
- 新 `Case` 实体不再包含 `cid`（文章分类ID），案例使用自己的 `industry_category` + `region_category` + `project_type` 分类体系
- 权限注解 `hasAuthority('admin:article:*')` 暂时保持不变，后续可改为 `admin:case:*`
- Admin 前端请求字段将从嵌套 `caseInfo: { projectName: ... }` 变为平铺 `projectName: ...`
