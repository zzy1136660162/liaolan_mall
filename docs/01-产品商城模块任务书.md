## 零、任务书定位

本任务书面向“只负责编码和结果确认”的开发人员。

开发人员职责仅包括：

- 按本任务书约定完成后端、后台、前台代码实现
- 按本任务书约定的数据库设计基线实现表结构或扩展结构
- 按本任务书约定的接口协议完成联调
- 按本任务书约定完成自测并提交结果

开发人员不负责：

- 重新设计商品业务模型
- 重新设计数据库方案
- 自行决定页面排版结构
- 自行增加筛选条件、状态枚举、业务字段
- 自行重构现有商城主链路

## 一、模块目标

在现有 CRMEB 商城底座上完成电线电缆行业化商品能力，包括：

- 多级分类商品展示
- 行业参数展示
- SKU 规格选型
- 加购、下单、支付
- 订单履约增强
- 会员中心企业信息增强

本模块是“基于现有商城二开”，不是重做商城。

## 五、数据库设计基线

### 5.1 数据库设计原则

- `eb_store_product` 继续作为商品主表
- 商品图文详情继续使用现有 `content`
- 行业参数不得继续堆进富文本
- SKU 继续复用现有规格表
- 履约扩展字段单独做扩展表，避免污染订单主表

### 5.2 建议表说明

| 表名 | 中文名 | 表用途 | 表类型 | 说明 |
|---|---|---|---|---|
| `eb_store_product_industry` | 商品行业扩展表 | 保存电线电缆行业结构化参数 | 扩展表 | 与 `eb_store_product` 一对一 |
| `eb_store_order_delivery_extend` | 订单履约扩展表 | 保存发货补充字段 | 扩展表 | 与 `eb_store_order` 一对一 |

### 5.3 字段说明

#### 表：`eb_store_product_industry`

| 字段名 | 中文含义 | 建议类型 | 必填 | 默认值 | 字段说明 |
|---|---|---|---|---|---|
| `id` | 主键 ID | `bigint` | 是 | 自增 | 主键 |
| `product_id` | 商品主键 ID | `bigint` | 是 | 无 | 关联 `eb_store_product.id` |
| `model_no` | 产品型号 | `varchar(128)` | 否 | 空字符串 | 如 `YJV`、`KVV` |
| `voltage_level` | 电压等级 | `varchar(64)` | 否 | 空字符串 | 如 `0.6/1kV` |
| `conductor_material` | 导体材质 | `varchar(64)` | 否 | 空字符串 | 如铜、铝 |
| `core_count` | 芯数 | `varchar(64)` | 否 | 空字符串 | 兼容多种表达，如 `3+1` |
| `cross_section_area` | 截面积 | `varchar(64)` | 否 | 空字符串 | 如 `70mm2` |
| `sheath_material` | 护套材质 | `varchar(64)` | 否 | 空字符串 | 如 PVC、PE |
| `flame_retardant_grade` | 阻燃等级 | `varchar(64)` | 否 | 空字符串 | 如 ZR、NH |
| `standard_code` | 执行标准 | `varchar(255)` | 否 | 空字符串 | 如 `GB/T 12706.1` |
| `application_scene` | 适用场景 | `varchar(255)` | 否 | 空字符串 | 如厂区供电、控制回路 |
| `certificate_files` | 证书文件集合 | `text` | 否 | 空 | 关联证书文件，建议存 JSON 数组 |
| `download_file_ids` | 关联资料 ID 集合 | `text` | 否 | 空 | 关联资料下载中心文件，建议存 JSON 数组 |
| `created_at` | 创建时间 | `datetime` | 是 | 当前时间 | 记录创建时间 |
| `updated_at` | 更新时间 | `datetime` | 是 | 当前时间 | 记录最后更新时间 |

约束要求：

- `product_id` 唯一
- 不允许复制商品主表中的名称、价格、库存字段
- `certificate_files`、`download_file_ids` 如存 JSON，前后端必须统一格式

#### 表：`eb_store_order_delivery_extend`

| 字段名 | 中文含义 | 建议类型 | 必填 | 默认值 | 字段说明 |
|---|---|---|---|---|---|

| `delivery_remark` | 发货备注 | `varchar(255)` | 否 | 空字符串 | 后台发货说明 |

## 八、详细开发内容

### 8.1 商品分类

开发要求：

- 复用现有分类树能力
- 支持按行业分类录入商品
- 前台分类展示不改变现有主链路

### 8.2 行业参数

开发要求：

- 商品新增 / 编辑页可维护行业结构化参数
- 商品详情页展示行业参数
- 不允许只存富文本不落结构化字段

### 8.3 SKU 规格

开发要求：

- 复用现有 SKU 机制
- 对长度、型号、芯数、截面积等可销售维度做规格化录入
- 非销售维度不得强行进入 SKU

### 8.4 下单支付

开发要求：

- 保持现有加购、预下单、创建订单、支付路径不变
- 支付成功后订单状态正常流转

### 8.5 订单履约

开发要求：

- 后台发货时支持维护物流公司、物流单号、发货备注
- 如业务要求，支持维护批次号和出厂编号

### 8.6 会员中心增强

开发要求：

- 在现有用户资料基础上增加企业信息字段
- 地址、订单记录链路不重构
