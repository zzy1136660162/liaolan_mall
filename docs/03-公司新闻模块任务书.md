# 公司新闻模块任务书

生成日期：2026-05-07  
适用项目：`E:\ZZY_PROJECT\crmeb_java`

## 零、任务书定位

本模块开发人员只负责编码和结果确认，不负责重新定义新闻运营体系。

## 一、模块目标

复用现有文章系统，建设企业新闻 / 行业资讯 / 展会动态发布能力。

## 二、项目现状与复用基线

可复用能力：

- 后台文章管理：`admin/src/views/content/article/*`
- 后台文章接口：`admin/src/api/article.js`
- 前台新闻页面：`app/pages/news/*`
- 前台文章接口：`app/api/api.js`
- 后端文章接口：`/api/front/article/*`

结论：

- 主体内容完全复用文章系统
- 仅做分类、页面与 SEO 扩展

## 三、负责人边界

负责人负责：

- 新闻分类落地
- 新闻前后台页面优化
- SEO 扩展字段开发

负责人不负责：

- 全站 SEO 总方案
- 内容运营录入

## 四、前置输入物料

- 新闻分类最终版
- 是否启用 SEO 字段
- 是否启用分享字段

## 五、数据库设计基线

### 5.1 原则

- `eb_article` 继续作为新闻主表
- SEO / 分享扩展字段单独扩展

### 5.2 建议表说明

| 表名 | 中文名 | 表用途 | 表类型 | 说明 |
|---|---|---|---|---|
| `eb_article_seo_extend` | 文章 SEO 扩展表 | 保存新闻 SEO 和分享字段 | 扩展表 | 与 `eb_article` 一对一 |

### 5.3 字段说明

#### 表：`eb_article_seo_extend`

| 字段名 | 中文含义 | 建议类型 | 必填 | 默认值 | 字段说明 |
|---|---|---|---|---|---|
| `id` | 主键 ID | `bigint` | 是 | 自增 | 主键 |
| `article_id` | 文章主键 ID | `bigint` | 是 | 无 | 关联 `eb_article.id` |
| `seo_title` | SEO 标题 | `varchar(255)` | 否 | 空字符串 | 页面 SEO 标题 |
| `seo_keywords` | SEO 关键词 | `varchar(255)` | 否 | 空字符串 | 页面 SEO 关键词 |
| `seo_description` | SEO 描述 | `varchar(500)` | 否 | 空字符串 | 页面 SEO 描述 |
| `share_title` | 分享标题 | `varchar(255)` | 否 | 空字符串 | 分享场景标题 |
| `share_image` | 分享封面 | `varchar(255)` | 否 | 空字符串 | 分享封面图 |
| `created_at` | 创建时间 | `datetime` | 是 | 当前时间 | 创建时间 |
| `updated_at` | 更新时间 | `datetime` | 是 | 当前时间 | 更新时间 |

## 六、UI 一致性要求

后台：

- 沿用现有文章编辑页
- SEO 字段位于内容配置后部
- 不改后台视觉体系

前台：

- 列表页继续复用现有新闻列表结构
- 标题、摘要、时间、封面位置统一
- 详情页继续复用当前阅读结构

## 七、API 设计基线

### 7.1 后台接口

| 接口路径 | 中文名称 | 端别 | 方法 | 用途说明 |
|---|---|---|---|---|
| `/admin/article/list` | 新闻列表 | admin | `GET/POST` | 获取后台文章列表 |
| `/admin/article/save` | 保存新闻 | admin | `POST` | 保存文章和 SEO 扩展字段 |
| `/admin/article/update` | 更新新闻 | admin | `POST` | 更新文章和 SEO 扩展字段 |
| `/admin/article/info` | 新闻详情 | admin | `GET` | 获取文章及 SEO 扩展字段 |

### 7.2 前台接口

| 接口路径 | 中文名称 | 端别 | 方法 | 用途说明 |
|---|---|---|---|---|
| `/api/front/article/category/list` | 新闻分类列表 | front | `GET` | 获取分类 |
| `/api/front/article/list/{cid}` | 新闻列表 | front | `GET` | 获取分类下新闻 |
| `/api/front/article/info` | 新闻详情 | front | `GET` | 获取新闻详情与 SEO 扩展字段 |

### 7.3 关键参数说明

| 参数名 | 中文含义 | 类型 | 必填 | 来源 | 默认值 | 说明 |
|---|---|---|---|---|---|---|
| `cid` | 分类 ID | `int` | 是 | `path` / `body` | 无 | 新闻分类 |
| `title` | 标题 | `string` | 是 | `body` | 无 | 新闻标题 |
| `synopsis` | 摘要 | `string` | 否 | `body` | 空字符串 | 新闻摘要 |
| `content` | 正文 | `string` | 否 | `body` | 空字符串 | 富文本正文 |
| `seoInfo` | SEO 扩展对象 | `object` | 否 | `body` | 空对象 | SEO 与分享字段 |

### 7.4 响应字段说明

| 字段名 | 中文含义 | 类型 | 必返 | 说明 |
|---|---|---|---|---|
| `id` | 新闻 ID | `int` | 是 | 主键 |
| `title` | 标题 | `string` | 是 | 新闻标题 |
| `imageInput` | 封面图 | `string` | 是 | 封面图地址 |
| `synopsis` | 摘要 | `string` | 是 | 摘要 |
| `content` | 正文 | `string` | 是 | 富文本 HTML |
| `seoInfo` | SEO 信息 | `object` | 是 | SEO / 分享字段 |
