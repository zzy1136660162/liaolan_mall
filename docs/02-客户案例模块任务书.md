# 客户案例模块任务书

生成日期：2026-05-07  
适用项目：`E:\ZZY_PROJECT\crmeb_java`

## 零、任务书定位

本模块开发人员只负责编码和结果确认，不负责重新设计案例内容体系。

## 一、模块目标

基于现有文章系统建设客户案例中心，实现：

- 案例列表
- 按行业 / 区域 / 项目类型筛选
- 案例详情结构化展示
- 关联产品跳转
- 引导询盘

## 二、项目现状与复用基线

当前项目已存在：

- 后台文章管理：`admin/src/views/content/article/*`
- 后台文章接口：`admin/src/api/article.js`
- 前台文章 / 新闻接口：`app/api/api.js`
- 前台内容页：`app/pages/news/*`
- 后端前台文章控制器：`ArticleController`
- 已存在文章绑定商品接口：`/admin/article/bind/product`

结论：

- 案例主体内容复用文章体系
- 案例结构化信息做扩展表

## 三、负责人边界

负责人负责：

- 案例扩展模型
- 后台案例发布页扩展
- 前台案例列表 / 详情页
- 案例与商品关联能力

负责人不负责：

- 商品详情页本体改造
- 通用留言模块开发

## 四、前置输入物料

- 案例分类字典最终版
- 案例详情字段清单最终版
- 是否支持多商品关联
- 询盘入口跳转策略

## 五、数据库设计基线

### 5.1 设计原则

- `eb_article` 继续作为案例正文主表
- 结构化案例字段单独扩展
- 不允许把区域、行业、项目类型全部塞入富文本

### 5.2 建议表说明

| 表名 | 中文名 | 表用途 | 表类型 | 说明 |
|---|---|---|---|---|
| `eb_article_case_extend` | 案例文章扩展表 | 保存案例结构化字段 | 扩展表 | 与 `eb_article` 一对一 |

### 5.3 字段说明

#### 表：`eb_article_case_extend`

| 字段名 | 中文含义 | 建议类型 | 必填 | 默认值 | 字段说明 |
|---|---|---|---|---|---|
| `id` | 主键 ID | `bigint` | 是 | 自增 | 主键 |
| `article_id` | 文章主键 ID | `bigint` | 是 | 无 | 关联 `eb_article.id` |
| `project_name` | 项目名称 | `varchar(255)` | 是 | 空字符串 | 展示项目名称 |
| `industry_category` | 行业分类 | `varchar(128)` | 否 | 空字符串 | 如电力、地产、市政 |
| `region_category` | 区域分类 | `varchar(128)` | 否 | 空字符串 | 如东北、华北 |
| `project_type` | 项目类型 | `varchar(128)` | 否 | 空字符串 | 如厂房、园区、变电项目 |
| `project_address` | 项目地点 | `varchar(255)` | 否 | 空字符串 | 具体地点或区域 |
| `project_period` | 项目周期 | `varchar(128)` | 否 | 空字符串 | 如 2025Q1、6 个月 |
| `project_background` | 项目背景 | `varchar(500)` | 否 | 空字符串 | 简述项目背景 |
| `supply_products` | 供货产品说明 | `varchar(500)` | 否 | 空字符串 | 简述供货产品 |
| `implementation_result` | 实施效果 | `varchar(500)` | 否 | 空字符串 | 案例结果摘要 |
| `product_ids` | 关联商品 ID 集合 | `text` | 否 | 空 | 建议存 JSON 数组 |
| `cover_images` | 案例图片集合 | `text` | 否 | 空 | 建议存 JSON 数组 |
| `sort` | 排序值 | `int` | 否 | 0 | 列表排序 |
| `created_at` | 创建时间 | `datetime` | 是 | 当前时间 | 创建时间 |
| `updated_at` | 更新时间 | `datetime` | 是 | 当前时间 | 更新时间 |

约束要求：

- `article_id` 唯一
- `product_ids` 为商品主键集合，不重复存商品名称

## 六、UI 一致性要求

### 6.1 后台

- 复用现有文章编辑风格
- 富文本继续使用现有 `Tinymce`
- 结构化案例字段位于正文配置区前部

### 6.2 前台

- 列表页延续新闻列表的阅读节奏
- 详情页顺序固定：
  1. 封面图
  2. 项目名称
  3. 项目结构化信息
  4. 关联产品入口
  5. 富文本正文
  6. 联系我们入口

## 七、API 设计基线

### 7.1 后台接口

| 接口路径 | 中文名称 | 端别 | 方法 | 用途说明 |
|---|---|---|---|---|
| `/admin/article/list` | 文章列表 | admin | `GET/POST` | 后台文章列表，支持案例分类使用 |
| `/admin/article/save` | 保存文章 | admin | `POST` | 保存案例文章及扩展字段 |
| `/admin/article/update` | 更新文章 | admin | `POST` | 更新案例文章及扩展字段 |
| `/admin/article/info` | 文章详情 | admin | `GET` | 返回案例文章及扩展字段 |
| `/admin/article/bind/product` | 文章绑定商品 | admin | `POST` | 维护案例关联商品 |

### 7.2 前台接口

| 接口路径 | 中文名称 | 端别 | 方法 | 用途说明 |
|---|---|---|---|---|
| `/api/front/article/list/{cid}` | 案例列表 | front | `GET` | 获取案例列表 |
| `/api/front/article/info` | 案例详情 | front | `GET` | 获取案例详情与扩展字段 |

| `synopsis` | 摘要 | `string` | 否 | `body` | 空字符串 | 案例摘要 |
| `content` | 富文本正文 | `string` | 否 | `body` | 空字符串 | 图文详情 |
| `caseInfo` | 案例扩展信息 | `object` | 否 | `body` | 空对象 | 结构化字段 |

| `projectBackground` | 项目背景 | `string` | 否 | `body` | 空字符串 | 背景简介 |
| `supplyProducts` | 供货产品 | `string` | 否 | `body` | 空字符串 | 供货说明 |
