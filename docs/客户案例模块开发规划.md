## 1. 模块概述

### 1.1 模块目标

基于现有 CRMEB 文章系统，建设客户案例中心，实现：
- 案例列表展示与筛选（行业 / 区域 / 项目类型）
- 案例详情结构化展示
- 案例与商品关联跳转
- 引导用户询盘

### 1.2 原型设计说明

参考 `docs/stitch_/_1/code.html` 的客户案例原型设计，页面包含：
- **分类标签栏**：横向滚动的行业分类标签（能源领域、建筑行业、电信通讯、轨道交通等）
- **标题区域**："工业应用案例"主标题和描述文字
- **案例卡片列表**：封面图（含标签）、项目名称、供货时长、工程规模、主要供货产品标签、"查看项目详情"按钮

### 1.3 项目现状与复用基线

**已有能力：**
- 后台文章管理：`admin/src/views/content/article/*`
- 后台文章接口：`admin/src/api/article.js`
- 前台文章接口：`app/api/api.js`
- 前台新闻页：`app/pages/news/*`
- 后端前台文章控制器：`ArticleController`
- 已存在文章绑定商品接口：`/admin/article/bind/product`

**复用策略：**
- 案例主体内容复用文章体系（`eb_article`）
- 案例结构化信息新建扩展表（`eb_article_case_extend`）
- 列表页参考原型设计结构

## 2. 数据库设计

### 2.1 设计原则

- `eb_article` 继续作为案例正文主表
- 结构化案例字段单独创建扩展表
- 不允许把区域、行业、项目类型全部塞入富文本

### 2.2 案例扩展表

**表名：** `eb_article_case_extend`

**字段说明：**

| 字段名 | 中文含义 | 类型 | 必填 | 默认值 | 说明 |
|--------|---------|------|------|--------|------|
| `id` | 主键 ID | `bigint` | 是 | 自增 | 主键 |
| `article_id` | 文章主键 ID | `bigint` | 是 | 无 | 关联 `eb_article.id`，唯一约束 |
| `project_name` | 项目名称 | `varchar(255)` | 是 | 空字符串 | 展示项目名称 |
| `industry_category` | 行业分类 | `varchar(128)` | 否 | 空字符串 | 如电力、地产、市政 |
| `region_category` | 区域分类 | `varchar(128)` | 否 | 空字符串 | 如东北、华北 |
| `project_type` | 项目类型 | `varchar(128)` | 否 | 空字符串 | 如厂房、园区、变电项目 |
| `project_address` | 项目地点 | `varchar(255)` | 否 | 空字符串 | 具体地点或区域 |
| `project_period` | 项目周期 | `varchar(128)` | 否 | 空字符串 | 如 2025Q1、6 个月 |
| `project_background` | 项目背景 | `varchar(500)` | 否 | 空字符串 | 简述项目背景 |
| `supply_products` | 供货产品说明 | `varchar(500)` | 否 | 空字符串 | 简述供货产品 |
| `implementation_result` | 实施效果 | `varchar(500)` | 否 | 空字符串 | 案例结果摘要 |
| `product_ids` | 关联商品 ID 集合 | `text` | 否 | 空 | 存储 JSON 数组格式 |
| `cover_images` | 案例图片集合 | `text` | 否 | 空 | 存储 JSON 数组格式 |
| `sort` | 排序值 | `int` | 否 | 0 | 列表排序 |
| `created_at` | 创建时间 | `datetime` | 是 | 当前时间 | 创建时间 |
| `updated_at` | 更新时间 | `datetime` | 是 | 当前时间 | 更新时间 |

## 3. API 设计

### 3.1 后台接口

| 接口路径 | 中文名称 | 端别 | 方法 | 用途说明 |
|---------|---------|------|------|---------|
| `/admin/article/list` | 文章列表 | admin | GET/POST | 后台文章列表，支持案例分类筛选 |
| `/admin/article/save` | 保存文章 | admin | POST | 保存案例文章及扩展字段 |
| `/admin/article/update` | 更新文章 | admin | POST | 更新案例文章及扩展字段 |
| `/admin/article/info` | 文章详情 | admin | GET | 返回案例文章及扩展字段 |
| `/admin/article/bind/product` | 文章绑定商品 | admin | POST | 维护案例关联商品 |

### 3.2 前台接口

| 接口路径 | 中文名称 | 端别 | 方法 | 用途说明 |
|---------|---------|------|------|---------|
| `/api/front/article/list/{cid}` | 案例列表 | front | GET | 获取案例列表，支持行业/区域/类型筛选 |
| `/api/front/article/info` | 案例详情 | front | GET | 获取案例详情与扩展字段 |

### 3.3 请求参数说明

#### 案例列表接口参数

```
GET /api/front/article/list/{cid}

Query 参数：
- industryCategory: 行业分类（可选）
- regionCategory: 区域分类（可选）
- projectType: 项目类型（可选）
- page: 页码（默认1）
- limit: 每页数量（默认10）
```

### 10.2 技术要点

1. **JSON 字段处理**
   - `product_ids` 和 `cover_images` 使用 JSON 数组格式
   - 后端使用 Jackson 进行序列化/反序列化

2. **联系我们入口**
   - 只需跳转到现有的联系我们页面，不开发新的 tabbar
   - 使用 `uni.navigateTo` 或已配置的路由跳转

3. **样式隔离**
   - 案例页样式使用独立的 class 前缀（如 `.case-`）
   - 避免样式污染全局

## 11. 附录

### 11.1 SQL 建表脚本

```sql
CREATE TABLE `eb_article_case_extend` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `article_id` bigint NOT NULL COMMENT '文章主键ID',
  `project_name` varchar(255) NOT NULL DEFAULT '' COMMENT '项目名称',
  `industry_category` varchar(128) NOT NULL DEFAULT '' COMMENT '行业分类',
  `region_category` varchar(128) NOT NULL DEFAULT '' COMMENT '区域分类',
  `project_type` varchar(128) NOT NULL DEFAULT '' COMMENT '项目类型',
  `project_address` varchar(255) NOT NULL DEFAULT '' COMMENT '项目地点',
  `project_period` varchar(128) NOT NULL DEFAULT '' COMMENT '项目周期',
  `project_background` varchar(500) NOT NULL DEFAULT '' COMMENT '项目背景',
  `supply_products` varchar(500) NOT NULL DEFAULT '' COMMENT '供货产品说明',
  `implementation_result` varchar(500) NOT NULL DEFAULT '' COMMENT '实施效果',
  `product_ids` text COMMENT '关联商品ID集合(JSON)',
  `cover_images` text COMMENT '案例图片集合(JSON)',
  `sort` int NOT NULL DEFAULT 0 COMMENT '排序值',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_article_id` (`article_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='案例文章扩展表';
```
